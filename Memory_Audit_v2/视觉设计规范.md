# 记忆档案馆 · Memory Archive — 视觉设计规范

## 一、品牌定位

| 维度 | 关键词 |
|------|--------|
| 情感基调 | 温暖、陪伴、治愈、浪漫、诗意、希望、梦想 |
| 品牌调性 | 轻奢、高级、克制、沉浸、故事感、长期品牌视觉 |
| 品质对标 | Apple 官网、Awwwards 获奖站点、Behance 精选、Dribbble 热门、电影级概念设计 |

整体传达「记忆的温热」而非技术的冰冷。用户进入页面应感受到被陪伴、被记得，而非面对一个工具或后台。

## 二、配色体系

### 背景系统

| 色阶 | 色值 | 用途 |
|------|------|------|
| 深夜蓝 | `#0a0e1a` | 页面最底层背景 |
| 深靛蓝 | `#0d1228` | 天空上半部 |
| 靛紫 | `#1a1a3e` | 天空中部过渡 |
| 午夜紫 | `#1f1535` | 天空下半部 |

背景采用多层径向渐变叠加，柔和自然过渡，避免生硬色块边界。

### 光源与点缀

| 色阶 | 色值 | 用途 |
|------|------|------|
| 暖金 | `#d4a853` | 主要发光元素、按钮渐变、高亮文字 |
| 蜜桃金 | `#e8c87a` | 光晕、内发光 |
| 淡金 | `#f0d89c` | 柔和高光、标题强调 |

### 辅助色

| 色阶 | 色值 | 用途 |
|------|------|------|
| 淡紫 | `#b8a9d4` | 粒子、柔光 |
| 粉云 | `#d4b8c8` | 雾气、云霞 |

### 文字色

| 色阶 | 色值 | 用途 |
|------|------|------|
| 暖白 | `#f0ece4` | 标题、主要文字 |
| 柔灰 | `#c8c4d0` | 正文 |
| 低对比灰 | `#8a8698` | 辅助信息、时间戳 |

### 配色原则

- 整体低饱和、高级灰度控制
- 渐变自然柔和，避免高饱和、高对比
- 暖金色仅用于强调和光源，不超过画面 5%
- 所有发光元素柔和发散，无刺眼高光

## 三、光影语言

| 技法 | 实现方式 |
|------|----------|
| Bloom 柔光 | 光源外围大范围低透明度径向渐变扩散 |
| 体积光 | 雾层半透明叠加，模拟空气散射 |
| 空气透视 | 远景降低对比度、增加蓝紫偏移 |
| 轮廓光 | 人物/物体边缘柔光描边 |
| 暖色环境光 | 整体暖色倾向的径向渐变浮层 |
| 光晕扩散 | 亮星粒子外围 2-3 层递减透明度光晕 |
| 粒子系统 | 40+ 漂浮粒子，缓慢上升，淡入淡出 |

### 光影原则

- 所有光源柔和发散
- 不出现刺眼高光和硬阴影
- 阴影仅用于 UI 卡片层级区分，不用`box-shadow`重阴影
- 卡片阴影：`0 2px 24px rgba(0,0,0,0.3)` + 内发光 `inset 0 0 0 1px rgba(255,255,255,0.04)`

## 四、插画风格

### 风格定义

半写实数字绘画（Semi-realistic Painterly Illustration），参考高品质动画概念设计。

### 元素清单

| 元素 | 说明 |
|------|------|
| 夜空 | 深蓝→靛紫→午夜紫渐变，180 颗闪烁星点 |
| 星体 | 大型发光天体（暖金渐变光晕），月牙形态 |
| 远山 | 低对比度剪影，蓝紫偏移 |
| 雾层 | 3 层水平半透明雾带，模拟空气透视 |
| 人物 | 比例较小（约占画面高度 8%），背影或侧影，位于画面中下方 |
| 水面/地面 | 低反射，深色渐变 |

### 禁止

- 动漫风格、卡通风格、写实摄影、赛博朋克
- 人物正面、人物特写、人物大比例
- 霓虹灯、机械、电路板、数据可视化元素

## 五、版式布局

### Hero Banner

```
┌──────────────────────────────────────────────┐
│  [Logo]                     [导航链接]       │
│                                              │
│  ┌──────────┐          ┌──────────────────┐ │
│  │          │          │                  │ │
│  │ 标题     │          │                  │ │
│  │ 副标题   │   30%    │   大型插画场景  70%│ │
│  │ 搜索框   │          │                  │ │
│  │ 主按钮   │          │                  │ │
│  │          │          │                  │ │
│  └──────────┘          └──────────────────┘ │
│                                              │
│  ┌─ 记忆卡片 ─┐  ┌─ 记忆卡片 ─┐  ...       │
│  └────────────┘  └────────────┘             │
└──────────────────────────────────────────────┘
```

### 布局原则

- 插画占约 70% 视觉面积，UI 占约 30%
- 大量留白，非对称平衡构图
- 阅读路径：左上 Logo → 标题 → 搜索框 → 结果卡片，Z 字形自然流畅
- 避免信息堆叠，每个视口聚焦一个核心操作
- 卡片网格：`minmax(360px, 1fr)`，自适应列数

### 间距系统（8pt）

| Token | 值 | 用途 |
|-------|----|------|
| xs | 8px | 图标间距、标签内边距 |
| sm | 16px | 卡片内边距、元素间距 |
| md | 24px | 卡片 padding、导航间距 |
| lg | 40px | section 内间距 |
| xl | 64px | hero 间距 |
| 2xl | 96px | hero 左右区域间距 |

## 六、UI 组件规范

### 6.1 玻璃拟态卡片（Glassmorphism）

```
background: rgba(20, 18, 40, 0.55)
border: 1px solid rgba(255, 255, 255, 0.06)
border-radius: 24px
backdrop-filter: blur(16px)
顶部 1px 渐变线: transparent → rgba(255,255,255,0.06) → transparent
hover: translateY(-4px) + 边框亮度提升 + 阴影加深
```

### 6.2 搜索框

```
胶囊形 (border-radius: 100px)
背景: rgba(255,255,255,0.04)
边框: 1px solid rgba(255,255,255,0.08)
backdrop-filter: blur(20px)
focus: 边框变暖金色调 + 外发光 (0 0 60px rgba(212,168,83,0.12))
```

### 6.3 按钮（主按钮）

```
胶囊形 (border-radius: 100px)
渐变: linear-gradient(135deg, #c8964a 0%, #d4a853 40%, #e0bc6a 100%)
内发光: inset 0 0 0 1px rgba(255,220,160,0.2)
外发光: 0 0 20px rgba(200,150,70,0.25)
文字: 深色 #1a1220
hover: 外发光加强 + translateY(-1px)
```

### 6.4 标签（Badge）

```
胶囊形 (border-radius: 100px)
字号: 11px
字距: 0.04em
半透明背景 + 柔和配色
- 已确认: rgba(100,200,140,0.12) → #8cd4a8
- 待审核: rgba(200,170,100,0.12) → #d4b870
- 已修正: rgba(140,140,200,0.12) → #b0b0d8
```

### 6.5 Tab 切换

```
胶囊形
默认: 透明背景 + 透明边框 + 灰色文字
hover: 边框可见 + 文字变亮
active: 玻璃背景 + 亮文字
```

## 七、字体系统

| 层级 | 字体 | 大小 | 字重 | 用途 |
|------|------|------|------|------|
| Hero 标题 | Georgia / Songti SC | 48px | 400 | 主标题 |
| Section 标题 | Georgia / Songti SC | 28px | 400 | 区块标题 |
| 卡片标题 | Georgia / Songti SC | 18px | 400 | 记忆卡片标题 |
| 正文 | PingFang SC / Microsoft YaHei | 14-16px | 400 | 描述文字 |
| UI 标签 | SF Pro Display / PingFang SC | 11-14px | 400-500 | 按钮、导航、标签 |
| 数据展示 | Georgia / Songti SC | 36px | 400 | 仪表盘数值 |

### 字体原则

- 标题优雅书卷气，使用衬线体（Georgia / 宋体）
- 正文轻盈现代，使用无衬线体（PingFang SC / 雅黑）
- 层级清晰，字距舒适（标题 0.02em，UI 0.04em）
- 禁止：未来科技字体、粗黑字体、等宽字体

## 八、动效规范

| 元素 | 动效 | 时长 | 缓动 |
|------|------|------|------|
| 背景星点 | 闪烁（正弦波透明度） | 3-8s 周期 | 持续 |
| 漂浮粒子 | 缓慢上升 + 水平微漂 | 生命周期 400-600 帧 | 持续 |
| 背景渐变 | 大半径缓慢位移 | 20-28s | ease-in-out |
| 卡片 hover | ↑4px + 阴影加深 | 0.35s | cubic-bezier(0.22, 1, 0.36, 1) |
| 搜索框 focus | 边框色变 + 外发光 | 0.35s | 同上 |
| 按钮 hover | 外发光 + ↑1px | 0.35s | 同上 |
| Toast | 底部滑入 + 淡入 | 0.35s | 同上 |
| Tab 切换 | 背景/边框色变 | 0.35s | 同上 |
| Logo 光点 | 脉冲光晕 | 3s | ease-in-out |

### 动效原则

- 统一缓动函数 `cubic-bezier(0.22, 1, 0.36, 1)` — 弹性但不跳脱
- 无自动播放的轮播、无弹跳、无旋转、无闪烁
- 所有动效服务于「静谧而富有生命力」的氛围

## 九、响应式策略

| 断点 | 布局变化 |
|------|----------|
| > 1024px | 完整 Hero 横排、4 列仪表盘、多列卡片网格 |
| 640-1024px | Hero 竖排、插画缩小至 320px 高、2 列仪表盘、单列卡片 |
| < 640px | 标题缩小至 32px、隐藏导航链接、1 列仪表盘、搜索按钮文字缩小 |

## 十、禁止清单

- 赛博朋克 / 霓虹灯 / 荧光色
- 游戏 UI 风格
- Material Design 卡片阴影和浮动按钮
- 企业后台 / 数据大屏风格
- 复杂信息布局 / 密集表格
- 厚重阴影 (`box-shadow` ≤ 0.4 透明度)
- 高饱和颜色（饱和度 ≤ 60%）
- 尖锐几何造型 / 直角
- 大量描边 / 边框宽度 ≤ 1px
- 机械科技感 / 电路板 / 芯片元素
- 过度装饰 / 花哨特效
- 硬阴影 / 刺眼高光
