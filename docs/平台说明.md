# 可信人格记忆Agent 平台说明

本文档面向第一次使用可信人格记忆Agent的开发者或评测人员，说明当前准备事项、启动方式、验证方式和常见结果。当前代码覆盖 Milestone 0 基础工程、健康检查、认证 API、免注册本地演示入口、Milestone 1 人物创建/工作台基础闭环、Milestone 2 资料上传/任务队列基础闭环、Milestone 3 mock 多模态解析和记忆审计基础闭环、Milestone 4 人格档案和可信度基础闭环、Milestone 5 第一人称文本对话基础闭环、Milestone 6 Task 1/2/3/4 后端默认 TTS、语音合成、音色样本、音色克隆兜底、语音消息 ASR 到 TTS、前端录音、声音设置和 mock 语音播放基础闭环，以及 Milestone 7 Task 1 后端 3D 数字人配置、默认形象和 mock 生成基础闭环；真实 OCR/ASR/VLM/模型质量、真实 LLM/embedding 检索质量、前端 3D/GLB 展示、故事/导出仍待后续里程碑接入。

## 适用对象与使用前准备

### 适用对象

- 开发者：验证本地基础工程、后端测试、前端检查、Compose 配置、Milestone 1 人物 API/页面基础闭环、Milestone 2 资料/任务 API 与页面基础闭环、Milestone 3 mock 解析/记忆 API 与页面基础闭环、Milestone 4 profile/trust API 与页面基础闭环、Milestone 5 text chat API 与页面基础闭环、Milestone 6 voice/default-tts/samples/clone/synthesize/voice-message API 与前端语音基础闭环，以及 Milestone 7 后端 avatar/default/generate API 基础闭环。
- 评测人员：确认当前是否具备前后端基础连通、健康检查、注册/登录 API、人物 CRUD、人物列表、人物创建表单校验、人物详情/工作台页、资料上传页、任务状态页、记忆审计页、人格档案/可信度页和文本对话页。
- 产品体验者：当前可以从中文首页点击「立即体验示例」免注册进入虚构人物「外婆」工作台，也可以使用注册/登录表单进入私有账号；人物创建/工作台、资料上传页、任务状态页、记忆审计页、人格档案/可信度页、文本对话页和声音设置页已中文化。上传资料会生成 deterministic mock 记忆卡片供确认、修改、拒绝、禁用或删除，审核后 profile/trust 会按 local deterministic 规则刷新，文本对话会使用 confirmed/corrected 记忆并展示引用依据；声音设置页可选择默认 TTS、创建音色样本、发起 mock 音色克隆和播放语音预览，聊天页可直接录音或用已上传音频发送 mock 语音消息并播放回复。3D 形象当前已有后端 mock API，暂不能在前端体验数字人、故事导出或真实模型质量。

### 使用前准备

当前需要具备：

- 能打开本仓库。
- Python 3.12 或兼容环境，可安装 `backend/requirements.txt`。
- Node.js 和 npm；在 PowerShell 中运行 npm 命令时使用 `npm.cmd`。
- Docker Desktop 或兼容 Docker Compose，用于验证 Compose 配置和本地服务拓扑。

不需要提交真实密钥。Compose 和后端配置只使用变量名、空白 provider 配置和开发默认值；真实 provider key 应只放在本地未提交环境中。

### 启动与访问

安装前端依赖：

```powershell
npm.cmd --prefix frontend install
```

运行统一 harness：

```powershell
& "C:\Program Files\Git\bin\bash.exe" ./docs/init.sh
```

启动本地服务拓扑：

```powershell
docker compose up --build
```

后端容器启动时会先执行 `alembic upgrade head`，再启动 Uvicorn 服务；因此 Compose 中的注册/登录 API 会使用已迁移的 PostgreSQL 表结构。

默认访问地址：

- 前端：http://localhost:3000
- 后端健康检查：http://localhost:8000/health
- MinIO Console：http://localhost:9001

## 用户全流程操作步骤

以下只描述当前可验证流程。完整人格记忆产品流程需在后续里程碑实现后补齐。

### 1. 进入系统

启动 Compose 后打开 http://localhost:3000 查看前端；访问 http://localhost:8000/health，返回 `{"status":"ok"}` 表示后端健康检查可用。首页主按钮「立即体验示例」会创建本地演示用户和虚构人物「外婆」，无需邮箱或密码；进入 `/login` 或 `/register` 可使用正式账号表单。

### 2. 配置或确认人格记忆范围

当前已实现人物创建基础配置。登录后进入 `/dashboard`，可查看当前账号的人物列表；进入 `/personas/new` 可创建已故亲友、在世亲友、公众人物或虚拟角色，并填写姓名、TA 对用户的角色、TA 对用户的称呼、说话风格、情绪陪伴方式和禁用表达。缺少必填字段时，前端会阻止提交。

### 3. 输入或导入记忆材料

当前已实现资料记录和 mock 解析。登录或进入演示后，从人物工作台点击「上传资料」进入 `/personas/{id}/uploads`，可上传文本、图片、音频、视频文件，也可填写手动资料。提交后系统创建 SourceMaterial 记录，把上传文件写入仓库本地 `storage/materials/...` 兜底路径，并通过 deterministic mock Provider Gateway 创建 `ParsedChunk` 和带来源的 `MemoryCard`，对应 AI Job 会完成为 `succeeded`。

### 4. 查看记忆提取结果

当前已实现 mock 记忆提取结果展示。进入 `/personas/{id}/memories` 可查看系统从资料中整理出的记忆卡片，并按状态、分类和置信度筛选；每条记忆展示标题、内容、来源摘录、来源位置、来源资料和置信度。进入 `/personas/{id}/jobs` 可查看资料对应的 AI Job 状态、任务类型、重试次数和错误信息；当前解析为同步 mock 输出，不代表真实 OCR、ASR、图片理解、视频分析或模型质量。

### 5. 复核、修正或删除记忆

当前已实现记忆复核、修正和删除操作。进入 `/personas/{id}/memories` 后，可以确认、编辑标题/内容、拒绝、禁用或删除记忆；编辑会保存为用户修正后的内容。Milestone 4 后这些状态会刷新人格档案和可信度；Milestone 5 后 confirmed/corrected 记忆会进入文本对话检索，rejected/disabled/deleted 记忆不会被引用。

### 6. 查看和编辑人格档案与可信度

当前已实现 deterministic local 人格档案和可信度页。进入 `/personas/{id}/profile` 可查看可信度总分、等级、PRD 权重组件、上传建议、profile summary、各档案维度和来源记忆 ID；页面支持保存手动档案修改、从已审核记忆重生成档案，以及手动重算可信度。显式重生成和重算会创建 succeeded `update_profile` 与 `calculate_trust_score` AI Job。

### 7. 在 Agent 交互中使用记忆

当前已实现文本对话中的基础记忆引用。进入 `/personas/{id}/chat` 后，页面会加载或创建当前人物的 conversation，用户可以发送文本消息；后端使用人物关系、称呼、说话风格、情绪风格、禁用表达、profile summary、对话历史和 confirmed/corrected 记忆生成第一人称 mock 回复。涉及具体记忆时，回复下方可展开依据，显示引用的记忆和来源位置；用户可在依据区纠正这条记忆，纠正后后续对话优先使用修正内容。

### 8. 使用默认 TTS（后端 API）

当前已实现后端默认 TTS 配置、mock 语音合成、音色样本和 mock 音色克隆 API。`GET /api/personas/{id}/voice` 可查看声音状态；初始状态为 `no_voice`。`POST /api/personas/{id}/voice/default-tts` 可选择系统默认 TTS，并返回提示：

```text
当前使用系统默认声音，不是 TA 的真实声音。上传 TA 的清晰语音后，可以尝试生成模拟音色。
```

`POST /api/personas/{id}/voice/samples` 可从当前用户当前人物的音频资料创建 `sample_ready` 音色样本，并写入 `extract_voice_sample` AI Job。`POST /api/personas/{id}/voice/clone` 可发起 mock `voice_clone`；成功时返回 selected `cloned_ready` 和 mock model URL，失败时标记 `clone_failed` 并回退 selected `default_tts`。`POST /api/personas/{id}/voice/synthesize` 会通过 mock `tts` Provider Gateway 返回 `mock://tts/...wav` 音频 URL，并创建 `synthesize_speech` AI Job。`POST /api/conversations/{id}/voice-message` 可从当前用户当前 conversation 下的音频资料执行 mock ASR、文本 chat 和 TTS，把 `audio_url` 写入 persona message；相关 `asr_audio` 与 `synthesize_speech` 任务可在 `/api/personas/{id}/jobs` 查看。前端 `/personas/{id}/voice` 可设置默认 TTS、创建样本、发起 mock 克隆并播放预览；聊天页可使用浏览器录音上传音频，也可选择已上传音频发送语音消息并播放返回音频。当前不代表真实 ASR/TTS/音色质量。

当前已实现后端 3D 数字人基础 API。`GET /api/personas/{id}/avatar` 可查看当前形象状态；初始状态为 `no_avatar`。`POST /api/personas/{id}/avatar/default` 可选择默认纪念形象，返回 mock GLB、预览图、基础表情、待机/眨眼/微笑/点头动作配置和 audio envelope 口型配置。`POST /api/personas/{id}/avatar/generate` 可从当前用户当前人物的图片资料创建 `avatar_3d` AI Job，通过 mock `avatar_3d` Provider Gateway 生成 `generated_ready` AvatarModel；生成失败时写入 `generation_failed` 并回退默认纪念形象，返回提示：

```text
这张照片暂时没有生成成功。你可以换一张更清晰的正脸照，或者先使用默认纪念形象继续对话。
```

当前不代表真实 3D 生成质量，也不包含前端 Three.js/GLB 加载、对话页数字人联动或口型同步可视化。

### 9. 导出或审计

当前未实现导出、审计回放或版本查看；后端已有 `AuditLog` 模型骨架供后续实现使用。

## 技术方案

当前已确认的技术方案：

- 前端：Next.js 15、React 19、TypeScript、Tailwind CSS；当前已接入 auth helper、persona helper、materials/jobs/memories/profile/chat helper、人物路由、资料上传页、任务状态页、记忆审计页、人格档案/可信度页、文本对话页和基础表单校验。
- 后端：FastAPI、SQLAlchemy、Alembic、JWT Auth、用户隔离的人物 CRUD、SourceMaterial API、AI Job API、mock parsing service、Memory Audit API、profile/trust API、text chat API 和 voice/default TTS/samples/clone/synthesize/voice-message API。
- 数据存储：PostgreSQL/pgvector、Redis、MinIO 通过 Docker Compose 接入；后端容器启动前执行 Alembic 迁移；测试使用 SQLite 内存库；当前上传文件使用仓库本地 `storage/materials/...` 作为开发兜底。
- AI 能力：当前通过 provider gateway 返回 deterministic mock parsing、memory extraction、`chat_llm`、`tts`、`extract_voice_sample` 和 `voice_clone` 输出；profile/trust 使用 deterministic local 规则；不接真实 OCR、ASR、VLM、embedding、LLM、TTS、voice clone 或 3D provider。
- 测试与部署：`docs/init.sh`、后端 pytest、前端 test/lint/build、`docker compose config`。
