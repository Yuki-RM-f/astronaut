# PRD 对照清单

本文档用于把 `docs/可信人格记忆Agent_mvp_prd.md` 中的产品要求与当前代码事实对齐。当前仓库已完成 Milestone 0-8 的后端/API 基础闭环、provider settings 后端运行配置 API、长期/短期记忆 Markdown、age 必填、Memory Audit v2 记忆档案馆、Persona Engine 显式画像分析和真实 provider adapter；当前前端以 `origin/yawen` 的星空交互为准，保留星空首页、Dashboard、创建星星页、人物详情、资料上传、任务、记忆档案馆、档案、对话、声音和 3D 页面，不再展示登录/注册、登录态差异、数据设置页、模型设置页或独立回忆讲述页。后端注册/登录、导出/删除、provider settings 和 stories API 保留不变；本文只记录已经存在的代码事实和仍待后续里程碑补齐的差距。

## 总体结论

- 当前结论：Milestone 0-8 的后端/API、`POST /api/auth/demo`、创建人物 age 必填、资料/任务、Memory Audit v2、profile/trust、Persona Engine 显式重生成、chat/voice/avatar/stories/export/delete/provider settings API 均保留；当前前端已完成 yawen 星空交互迁移并将 `/personas/{id}/memories` 升级为星空主题记忆档案馆。无 token 时前端通过 `ensureDemoSession()` 透明调用现有 demo session，已有 token 静默复用；创建星星页保留 `age` 和当前后端 payload 契约，创建成功后调用现有资料上传 API 上传已选文件。
- 当前风险：Memory Audit v2 本次只集成审计日志、冲突检测、语义搜索、记忆历史和 Persona Engine 显式画像分析，不实现 v1 snapshot、rollback 或 persona drift。yawen 未闭环的视频手势互动、遗憾对话室和心愿延续只作为前端模式或 prompt 入口。独立 `/settings/data`、`/settings/providers` 和 `/personas/{id}/stories` 页面当前不再暴露；对应后端 API 仍可通过接口使用或未来重新挂入口。当前前端依赖树仍有 2 个 moderate severity npm audit findings。
- 当前建议：后续里程碑继续在本清单中逐项补齐真实 LLM 长期质量、MiniMax TTS/音色端到端质量验收、真实 GLB/3D provider 质量、真实 MinIO/S3 对象删除和真实音频音量包络/viseme 口型同步质量；DashScope 真实解析全量公网样本 smoke 已在 2026-07-04 复跑通过，后续只需在模型或账号配置变化后做回归验证。

提供用户隔离的 story list/generate/favorite/export/audio-file export API；`backend/app/services/data_management.py` 复用 PRD 软删除规则供人物删除和账号数据清空使用；`backend/app/services/provider_settings.py` 生成 provider 输出、对话历史、禁用表达和模型 `<think>` 推理块过滤、`message_citations`、ASR AI Job 和 TTS audio URL 生成第一人称文本/语音消息回复，并支持单条 conversation 级联软删除 messages/citations；`backend/app/services/voice.py` 使用 `VoiceModel`、mock/可选
